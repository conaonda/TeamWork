# 코드리뷰 가이드

## 리뷰어 체크리스트

### 기능
- [ ] 요구사항을 충족하는가?
- [ ] 엣지 케이스가 처리되었는가?

### 코드 품질
- [ ] 읽기 쉽고 이해하기 쉬운가?
- [ ] 불필요한 중복이 없는가?
- [ ] 네이밍이 명확한가?

### 안전성
- [ ] 보안 취약점이 없는가? (XSS, SQL Injection 등)
- [ ] 에러 핸들링이 적절한가?
- [ ] 민감한 정보가 노출되지 않는가?

### 테스트
- [ ] 테스트가 포함되었는가?
- [ ] 기존 테스트가 통과하는가?

## 코멘트 접두어
사람과 에이전트 모두 동일한 접두어를 사용합니다:
- `[필수]` — 반드시 수정 필요
- `[제안]` — 선택적 개선 사항
- `[질문]` — 이해를 위한 질문

## 리뷰 응답 시간
- 에이전트 리뷰어: 즉시 처리
- 사람 리뷰어: 일반 PR 24시간 이내, 핫픽스 PR 가능한 빨리

---

## 사람 리뷰어 원칙
- 코드를 비판하되 사람을 비판하지 않습니다.
- 질문 형태로 피드백합니다 ("이 부분은 ~하면 어떨까요?").
- 좋은 코드에도 칭찬을 남깁니다.

## 에이전트 리뷰어 규칙

### 리뷰 출력 형식
```markdown
## 리뷰 결과: [승인/변경요청]

### 체크리스트
- [x] 요구사항 충족
- [x] 보안 취약점 없음
- [ ] 에러 핸들링 미흡 — 설명

### 코멘트
- [필수] `파일경로:라인번호` — 설명
- [제안] `파일경로:라인번호` — 설명
```

### 자동 체크 항목
에이전트 리뷰어는 다음을 자동으로 검사합니다:
- OWASP Top 10 보안 취약점
- 하드코딩된 시크릿/인증 정보
- 불필요한 의존성 추가
- 기존 유틸리티와 중복되는 구현
- 테스트 커버리지 감소
